# Геймдизайн-документ: «НИИ-2077: Заслуженный Кибернетик»

## 1. Концепция игры
**Жанр:** Нарративный симулятор жизни (swipe-based RPG в духе Reigns).
**Сеттинг:** СССР-ретро-футуризм. Вакуумные лампы, перфокарты, кибернетические импланты советского производства, мега-ЭВМ размером с хрущевку, талоны на нейростимуляторы, строгая бюрократия и построение «Кибер-Коммунизма».
**Завязка:** Главный герой — молодой специалист, выпускник кибернетического факультета, который получает распределение в НИИ Вычислительных Машин.
**Глобальная цель:** Выжить в бюрократическом аппарате, сделать карьеру, не сойти с ума, не попасть в застенки Кибер-КГБ и выйти на почетную пенсию (в идеале — на личной смарт-даче).
**Кор-луп (Core Loop):** Игроку подается карточка-ситуация. Он выбирает «Да» или «Нет». Выбор меняет параметры персонажа и скрытые флаги. Время (возраст/стаж) идет вперед.

---

## 2. Структура состояния персонажа (State)

Вместо сохранения истории шагов, игра отслеживает только текущий срез жизни главного героя.

### 2.1. Базовые параметры (Шкалы 0-100)
Если параметр падает до нуля или доходит до 100, могут триггериться критические события (например, увольнение или нервный срыв).

*   **Компетенция (Tech):** Навыки программирования, логика, умение работать с ЭВМ. Влияет на карьерный рост.
*   **Стресс (Stress):** Усталость от переработок («кранчей»), бюрократии и сложных задач.
*   **Здоровье (Health):** Физическое и ментальное. Сажается дешевыми нейростимуляторами и сидячим образом жизни. Восстанавливается в советских санаториях.
*   **Лояльность (Loyalty):** Отношение к Партии и руководству НИИ. Высокая — ты «удобный» партийный работник. Низкая — диссидент, склонный к кибер-панку.
*   **Авторитет (Reputation):** Уважение коллег, связи в коллективе.
*   **Сбережения (Money):** Талоны, рубли, крипто-рубли. Нужны для взяток, покупки техники на черном рынке, ухаживаний.

### 2.2. Трекеры и счетчики (Counters)
*   **Возраст / Стаж:** Главный таймер игры. Начинается с 22 лет (или 0 лет стажа), пенсия в 60 лет (38 лет стажа).
*   **Должность (Job Level):** Младший научный сотрудник (МНС) -> СНС -> Завлаб -> Директор направления.
*   **Уровень доступа (Clearance):** Открывает секретные проекты (и опасные события).

### 2.3. Флаги (Flags)
Бинарные состояния или короткие строки, описывающие ключевые факты прошлого:
*   `has_blackmarket_implant: true` (Поставил левый имплант).
*   `kgb_suspicion: true` (На карандаше у КГБ).
*   `family_status: "married_to_nina"` (Женат на коллеге).
*   `project_ogas_sabotaged: true` (Запорол важный проект).

## 3. Игровые Арки (Независимые линии)

Жизнь персонажа генерируется из пересечения нескольких арок. События в каждой арке зависят от параметров, но развиваются параллельно.

1.  **Арка Карьеры:** Рутинные задачи, баги на перфокартах, конфликты с начальством, рационализаторские предложения.
2.  **Арка Системы (Политика):** Собрания парткома, проверки Кибер-КГБ, предложения написать донос на коллегу, цензура кода.
3.  **Арка Быта и Кибернетики:** Жизнь в кибер-коммуналке, попытки достать дефицитные запчасти, покупка роботизированного Москвича, здоровье.
4.  **Арка Отношений:** Служебные романы, дружба с фарцовщиками, семейная жизнь, воспитание детей-пионеров.

## 4. Архитектура Событий (Event System)

В игре нет дерева сюжета. Есть «Колода» из сотен карточек. Карточки попадают в руку игроку на основе условий и вероятностей.

### Типы карточек (Событий):
*   **Повседневные (Repeatable):** Обычные рабочие моменты с кулдауном (чтобы не повторялись чаще раза в год).
*   **Триггерные (Conditional):** Появляются только если статы достигли определенной зоны (например, Стресс > 80).
*   **Уникальные (Once):** Крупные вехи, случаются только один раз (первое повышение, свадьба).

### Общая структура данных карточки (Концептуальный JSON):

```json
{
  "id": "black_market_ram",
  "category": "lifestyle",
  "text": "Знакомый фарцовщик предлагает купить контрабандный блок памяти для твоего рабочего терминала. С ним код будет компилироваться быстрее, но это незаконно. Берем?",
  "conditions": {
    "job_level": "МНС",
    "money_min": 20,
    "has_blackmarket_implant": false
  },
  "weight": {
    "base": 10,
    "modifier": "if loyalty < 40 then +10" // Чаще выпадает, если ты диссидент
  },
  "is_unique": true,
  "choices": {
    "yes": {
      "effects": {
        "money": -20,
        "tech": +15,
        "stress": -10,
        "loyalty": -5
      },
      "flags_set": {
        "has_blackmarket_implant": true
      }
    },
    "no": {
      "effects": {
        "loyalty": +5,
        "stress": +5
      }
    }
  }
}
```

## 5. Как работают последствия (Примеры механик)

### 5.1. Отложенные последствия через флаги
**Ситуация:** Игрок согласился написать донос на коллегу (Выбрал «Да»).
**Скрытый эффект:** Флаг `snitched_on_ivan = true`, Лояльность +20, Авторитет -5.
**Отложенное событие (появится через 10-20 ходов):**
*   *Условие карточки:* `snitched_on_ivan == true` и `reputation < 30`.
*   *Текст:* «Иван узнал, кто написал на него рапорт в партком. Он подстроил сбой в твоей программе прямо перед сдачей проекта Госплану».

### 5.2. Управление вероятностями (Веса)
Игрок выбирает «Да» на предложение: «Записаться в добровольную кибер-дружину?».
*   Это не переводит его в «ветку дружинника».
*   Это меняет веса: вероятность выпадения событий из пула «уличные драки с техно-панками», «поощрения от начальства» возрастает на 30%. Вероятность событий «посиделки в баре» падает.
*   *Ощущение игрока:* «Мой выбор изменил мой образ жизни».

### 5.3. Критические состояния (Выгорание)
Если игрок постоянно берет переработки (жмет «Да» на доп. задачи), растет Компетенция, но Стресс уходит за 90.
*   Движок принудительно достает карточку из пула "Кризис".
*   *Текст:* «У вас микроинсульт на рабочем месте. Система жизнеобеспечения вызывает скорую».
*   *Эффекты:* Пропуск 5 ходов, потеря Сбережений на лечение, понижение Стресса до 0, понижение Здоровья.

## 6. Макроуровень: Ключевые Вехи (Сюжет)

Чтобы игра не казалась бесконечной случайной песочницей, в систему внедряются "События-Вехи", которые игнорируют веса и выпадают строго при достижении определенного стажа/возраста.

1.  **Год 3. Проверка Госкомиссии.** Выживешь или нет — зависит от накопленной Компетенции и Лояльности. Определяет, станешь ли ты Старшим сотрудником или останешься на низовке.
2.  **Год 10. Проект "ОГАС-2".** Глобальная стройка сети. Герою предлагают должность Завлаба. Если Авторитет низкий, коллеги устроят саботаж.
3.  **Год 25. Инцидент "Красный Экран".** Глобальный сбой во всесоюзной ЭВМ. Поиск козла отпущения. Спасут связи (КГБ или Авторитет).
4.  **Год 38. Пенсионный Комитет.** Финал игры.

## 7. Финалы (Пенсия)

Конец игры генерируется на основе итогового состояния `state`. Нет предзаписанных 50 концовок, концовка собирается из блоков.

**Пример генерации финала:**
*   *База (от должности):* Директор НИИ / Рядовой кодер / Уволенный диссидент.
*   *Модификатор Здоровья:* Если < 30 — «Вы подключены к аппарату искусственной жизни». Если > 70 — «Каждое утро делаете кибер-зарядку».
*   *Модификатор Семьи:* Одинокая старость / Окружен внуками-пионерами.
*   *Статус в обществе (Лояльность):* Получил орден Трудового Красного Знамени / Живете в тени, обходя алгоритмы надзора.

---

## 8. Игровой цикл (Алгоритм работы движка)

1.  **Начало Хода (Месяц/Год):** Возраст увеличивается.
2.  **Пассивные эффекты:** Если здоровье низкое, стресс понемногу растет сам.
3.  **Сбор кандидатов:** Движок смотрит базу из (например) 300 карточек. Отбрасывает те, где не совпадают условия (`conditions`). Отбрасывает те, что на кулдауне.
4.  **Взвешенный рандом:** Из оставшихся 15-20 доступных карточек выбирается одна с учетом ее модификатора вероятности (`weight`).
5.  **Показ:** Игрок видит карточку и 2 варианта ответа.
6.  **Применение (Resolving):** Изменение параметров, запись новых флагов.
7.  **Проверка триггеров:** Не достиг ли какой-то стат критической отметки? Если да — следующее событие будет кризисным.
8.  Переход к шагу 1, пока не сработает событие "Пенсия".

---

### Почему такой дизайн идеален для разработки:
1.  **Бесконечная масштабируемость:** Можно выпустить игру со 100 карточками, а потом просто докидывать в JSON новые карточки пачками. Они органично вплетутся в жизнь игроков.
2.  **Легко балансировать:** Если игроки слишком быстро повышают зарплату, достаточно добавить пару карточек с налогами или бытовыми поломками, которые триггерятся при высоком уровне денег.
3.  **Использование ИИ:** Формат карточек с параметрами идеально подходит для того, чтобы генерировать контент с помощью ChatGPT (например: *"Напиши 10 событий для арки 'отношения' с условием 'низкий авторитет' и 'высокие деньги' в советском ретро-футуризме"*).
